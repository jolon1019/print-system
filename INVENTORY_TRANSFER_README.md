# 货权转让功能使用说明

## 概述
货权转让功能允许用户将自己单位下的货物转让给其他单位，管理员可以查看和管理所有货权转让记录。

## 功能特性
- ✅ 用户可以将自己单位下的货物转让给其他单位
- ✅ 支持搜索和筛选转让记录
- ✅ 完整的转让历史记录
- ✅ 管理员可以查看所有转让记录
- ✅ 管理员可以取消转让记录
- ✅ 支持导出转让记录为Excel文件
- ✅ 转让原因记录和操作人追踪

## 安装步骤

### 1. 执行数据库脚本
在 Supabase 数据库中执行 `inventory_transfer.sql` 脚本：

```bash
# 在 Supabase SQL 编辑器中执行
inventory_transfer.sql
```

该脚本会创建 `inventory_transfer` 表，用于记录货权转让历史。

### 2. 文件说明
- `inventory_transfer.sql` - 货权转让表创建脚本
- `inventory-transfer-service.js` - 货权转让服务（Node.js后端）
- `user-view.html` - 用户端页面（已更新，包含货权转让功能）
- `admin.html` - 管理员端页面（已更新，包含货权转让管理功能）

## 使用方法

### 用户端使用

#### 1. 登录系统
使用普通用户账号登录系统。

#### 2. 进入库存信息页面
在用户端页面中，点击"库存信息"标签页。

#### 3. 执行货权转让
1. 在库存信息表格中，找到要转让的货物
2. 点击该货物行的"转让"按钮（只有状态为"在库"的货物才能转让）
3. 在弹出的对话框中：
   - 查看批号和当前单位（不可修改）
   - 选择或输入接收单位
   - 填写转让原因（必填）
4. 点击"确认转让"按钮完成转让

#### 4. 查看转让记录
1. 点击"货权转让"标签页
2. 可以查看所有相关的转让记录（包括转出和转入）
3. 支持按批号、转让单位、接收单位搜索
4. 可以导出转让记录为Excel文件

### 管理员端使用

#### 1. 登录系统
使用管理员账号登录系统。

#### 2. 进入货权转让管理页面
1. 在管理员端页面中，点击"货权转让管理"标签页
2. 可以查看所有货权转让记录

#### 3. 查看转让详情
1. 在转让记录表格中，点击"详情"按钮
2. 在弹出的对话框中查看完整的转让信息和货物信息

#### 4. 取消转让记录
1. 在转让记录表格中，找到要取消的转让记录
2. 点击"取消"按钮（只有状态为"已完成"的转让才能取消）
3. 确认取消操作
4. 转让记录状态将变为"已取消"

#### 5. 导出转让记录
1. 点击"导出记录"按钮
2. 系统将生成包含所有转让记录的Excel文件

## 数据权限说明

### 用户权限
- 只能转让自己单位下的货物
- 只能查看与自己单位相关的转让记录（作为转让方或接收方）
- 不能取消转让记录

### 管理员权限
- 可以查看所有货权转让记录
- 可以取消任何转让记录
- 可以导出所有转让记录

## 数据库表结构

### inventory_transfer 表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | bigint | 转让ID（主键） |
| inventory_id | bigint | 库存ID（外键） |
| batch_no | varchar(50) | 批号 |
| from_unit | varchar(50) | 转让单位 |
| to_unit | varchar(50) | 接收单位 |
| transfer_date | timestamp | 转让时间 |
| transfer_reason | text | 转让原因 |
| operator_id | bigint | 操作人ID |
| operator_name | varchar(50) | 操作人姓名 |
| status | smallint | 状态：1-已完成，2-已取消 |
| remark | text | 备注 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

## 业务流程

### 货权转让流程
1. 用户选择要转让的货物（必须在库状态）
2. 填写转让信息（接收单位、转让原因）
3. 系统更新库存表的单位字段
4. 系统创建转让记录
5. 转让完成，货物所有权转移

### 取消转让流程
1. 管理员选择要取消的转让记录
2. 确认取消操作
3. 系统更新转让记录状态为"已取消"
4. 注意：取消转让不会恢复货物的原始单位

## 注意事项

1. **转让限制**
   - 只能转让状态为"在库"的货物
   - 不能将货物转让给当前单位
   - 转让原因必须填写

2. **数据安全**
   - 所有转让操作都会记录操作人和操作时间
   - 转让记录不可删除，只能取消
   - 取消转让不会恢复原始数据

3. **权限控制**
   - 普通用户只能转让自己单位下的货物
   - 只有管理员可以取消转让记录
   - 管理员可以查看所有转让记录

4. **数据一致性**
   - 转让操作会同时更新库存表和转让记录表
   - 确保数据库事务的完整性
   - 建议定期备份数据库

## 故障排除

### 问题：无法找到转让按钮
- 检查货物状态是否为"在库"
- 确认用户是否有该货物的转让权限
- 检查用户是否绑定了正确的单位

### 问题：转让失败
- 检查接收单位是否正确
- 确认转让原因是否填写
- 查看浏览器控制台是否有错误信息
- 检查数据库连接是否正常

### 问题：无法查看转让记录
- 确认用户是否绑定了相关单位
- 检查数据库中是否有转让记录
- 刷新页面重新加载数据

### 问题：管理员无法取消转让
- 检查转让记录状态是否为"已完成"
- 确认当前用户是否为管理员
- 查看数据库连接是否正常
